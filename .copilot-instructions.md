# OpenAGI Copilot Instructions

## Project Overview

OpenAGI is an open-source Artificial General Intelligence (AGI) framework designed to democratize access to advanced AI capabilities. This project aims to create a modular, scalable, and ethically-aligned AGI system that can be used for research, development, and practical applications.

## Core Principles

### 1. Safety First
- Always prioritize AI safety and alignment in code suggestions
- Implement robust error handling and fail-safes
- Consider potential misuse scenarios and implement appropriate safeguards
- Follow responsible AI development practices

### 2. Transparency and Explainability
- Code should be well-documented and self-explanatory
- AI decision-making processes should be traceable and auditable
- Maintain clear separation between AI logic and application logic

### 3. Modularity and Extensibility
- Design components to be loosely coupled and highly cohesive
- Use dependency injection and interface-based programming
- Enable easy integration of new AI models and capabilities

## Coding Standards

### General Guidelines
- Use clear, descriptive variable and function names
- Write self-documenting code with minimal but meaningful comments
- Follow SOLID principles and clean code practices
- Prefer composition over inheritance
- Implement proper error handling and logging

### Python Code Style (Primary Language)
```python
# Follow PEP 8 standards
# Use type hints for all function parameters and return values
# Example:
def process_prompt(prompt: str, model_config: Dict[str, Any]) -> AIResponse:
    """Process a user prompt through the AGI system.
    
    Args:
        prompt: The user input prompt
        model_config: Configuration parameters for the AI model
        
    Returns:
        AIResponse object containing the processed result
        
    Raises:
        ValidationError: If prompt is invalid
        ModelError: If model processing fails
    """
    pass
```

### JavaScript/TypeScript (for web interfaces)
- Use TypeScript for type safety
- Follow ESLint and Prettier configurations
- Use modern ES6+ features
- Implement proper async/await patterns

## Architecture Guidelines

### System Design
- Implement microservices architecture for scalability
- Use event-driven patterns for component communication
- Maintain clear boundaries between AI processing and business logic
- Design for horizontal scaling and distributed deployment

### AI/ML Components
- Separate model training, inference, and evaluation logic
- Implement model versioning and A/B testing capabilities
- Use containerization for model deployment
- Maintain clear data pipelines and preprocessing steps

### Data Handling
- Implement proper data validation and sanitization
- Use structured logging for all data operations
- Maintain data lineage and audit trails
- Follow privacy-by-design principles

## Security Considerations

### AI-Specific Security
- Implement input validation to prevent prompt injection attacks
- Use rate limiting and resource monitoring
- Sanitize all user inputs before processing
- Implement model output filtering and safety checks

### General Security
- Use secure authentication and authorization
- Implement proper secret management
- Follow OWASP security guidelines
- Regular security audits and vulnerability assessments

## Testing Requirements

### Unit Testing
- Maintain minimum 80% code coverage
- Test all public APIs and critical paths
- Use mocking for external dependencies
- Write tests for edge cases and error conditions

### AI/ML Testing
- Implement model validation and performance testing
- Test for bias and fairness in AI outputs
- Validate model behavior across different input types
- Monitor model drift and performance degradation

### Integration Testing
- Test end-to-end workflows
- Validate API contracts and data formats
- Test error handling and recovery scenarios

### Transparency

## Performance Guidelines

### Optimization
- Optimize for both latency and throughput
- Implement caching strategies for frequently accessed data

- Monitor and optimize resource usage

### Scalability
- Design for horizontal scaling
- Use load balancing and auto-scaling
- Implement circuit breakers for fault tolerance
- Monitor system health and performance metrics

## Dependencies and Libraries

### Preferred Technologies
- **AI/ML**: PyTorch, TensorFlow, Hugging Face Transformers
- **API Framework**: FastAPI, Flask (Python), Express.js (Node.js)
- **Database**: PostgreSQL, Redis (caching), Vector databases for embeddings
- **Message Queue**: RabbitMQ, Apache Kafka
- **Containerization**: Docker, Kubernetes
- **Monitoring**: Prometheus, Grafana, ELK Stack

### Dependency Management
- Pin dependency versions for reproducibility
- Regular security updates and vulnerability scanning
- Minimize dependencies and prefer well-maintained libraries
- Document all external dependencies and their purposes
